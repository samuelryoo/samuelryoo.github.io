Simon Game

Interactive visual/audio memorization game, using the AVR microcontroller (ATmega32).

C/C++ Embedded Systems Soldering KiCad

AVR-Based Embedded Memory System

For this project, I designed and implemented an interactive embedded system on an AVR microcontroller that integrates keypad input, LED indicators, an LCD display, and audio output. The objective was to build a responsive and deterministic system capable of coordinating multiple peripherals under tight timing and resource constraints, without relying on blocking delays.

The firmware is structured around a non-blocking main control loop, allowing input handling, display updates, and audio generation to be serviced continuously. Keypad input is debounced and processed asynchronously, while LCD rendering and tone generation are scheduled to avoid monopolizing CPU time. Game-state logic is separated from hardware drivers to preserve predictable execution and simplify future extension.

A primary engineering challenge was coordinating peripherals with differing timing requirements on a resource-constrained microcontroller. LCD updates, audio output, and input polling all compete for execution time, and naive implementations led to sluggish response or missed inputs. This was addressed by decomposing peripheral interactions into short, bounded operations and sequencing them deterministically within the main loop to maintain responsiveness.

The final system operates reliably with consistent input response and clear visual and audio feedback. The firmware architecture supports straightforward extension to additional difficulty levels or operating modes, demonstrating a scalable approach to embedded system design aligned with real-world firmware constraints.
